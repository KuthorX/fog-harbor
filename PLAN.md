# 《雾港在线 2000：零点调频》完整策划案

## 作品定位

- 类型：浏览器 ARG / 档案悬疑 / 中式微心理恐怖
- 时长：60—110 分钟
- 平台：桌面浏览器优先，手机单窗口适配
- 技术：原生 HTML/CSS/JavaScript，离线可玩
- 边界：无血腥、跳脸、追逐；不把真实谣言包装成事实

## 玩家契约

玩家恢复 2000 年元旦封存的校园网镜像。目标不是“找到十二条线索”，而是回答三个逐步变化的问题：北楼照片为何多出一人；N17 是否存在；邵雯是否失踪。

## 媒介矩阵

| 表面 | 叙事职责 | 谜题职责 |
|---|---|---|
| 雾港在线门户 | 建立时代与公共叙事 | 精确关键词入口 |
| Outlook | 私人动机与伪造材料 | 连锁信首字、机构名、临时文件名 |
| 我的文档 | 维护者视角 | 元素表、拨号频率、照片人数 |
| Winamp | 感官证据 | 摩斯码 ECHO |
| BBS | 传闻扩散与纠错 | N17 → N1-7 |
| 网上相册 / BP机 | 店员与设备记录 | 点击BP机读取寻呼数字，推导身份 |
| 个人主页 | 梁真的声线 | 视觉后像、N17、留言板 |
| 回收站 | 被主动删除的版本 | 观察员身份与第八人误读 |

## 主路径

`雾港之声 → 1702 → 槐荫路44号 → 蓝鲸照相馆 → 8010111 → LUMEN → N17 → 网页存档日期 → N1-7 → Winamp手动调频与选带 → ECHO → 听友来信 → QSL98516 → CASE_98 → 四份原件交叉复核 → ECHO结项摘要 → 返回磁盘中的SW文件夹 → 零点之后别回头`

每个硬门的答案至少来自两个表面：

- `8010111`：黄页规则 + 本地元素灯订单 + 外部周期表知识。
- `LUMEN`：相册 BP 数字 + 个人主页/邮件地址的身份验证。
- `ECHO`：Winamp 摩斯码 + BBS 管理员提及项目名。
- 最终口令：报告的回看指令 + 连锁信各段首字。

## 支线

- 千年虫：区分真实软件日期风险与硬件爆炸谣言。
- 镜子慢一秒：视觉后像、运动预测、暗示效应。
- POLYBIUS：展示论坛如何共同写成都市传说。
- 微波炉鸡蛋、仙人掌防辐射：将生活谣言放回可验证科学。

支线不阻塞结局，但为 ECHO 真相提供认知框架。

## 角色声线

- 梁真 / LUMEN：1998年许宁受伤当晚的广播站值机员。为给《夜航音乐》拉听众，他此前在BBS编过“午夜会叫到你的名字”的广告，转帖时玩笑说明被删。母带能证明主持人念的是徐玲，但他同时是传闻源头、当晚值机员和事故后监听副本的制作者，因此被怀疑换带。他加入ECHO是想证明错听来自传闻，不是自己放错带；这份急迫让他默许罗绍明使用真实姓名。
- 邵雯 / SW：实验观察员，也是梁真的朋友。她负责拍照、整理访谈和检查告知书。发现自己的试音被偷偷接入节目后退组；梁真的BBS帖子又让陌生人找到宿舍，她因此办完手续去南州。
- 罗绍明 / 罗盘：广播站指导老师。最初确实想弄清许宁事件，后来为了得到明显结果和会议论文，绕过重新签字，把观察员也当成材料。
- 纸月亮：参与者。只描述具体现象，不主动给解释，因此最容易被论坛引用。

## 结局节奏

实验报告先完成理性解释，给玩家短暂稳定感；邵雯留言随后改变情感中心：谜团从“谁消失了”变成“谁有权定义离开”。最后仅留下动态访问时间与未开启的收音机噪声，保留系列母谜团。

## 试玩验收

- 新玩家十分钟内能找到第一个有效检索词，但界面不直接标黄。
- 玩家能说明每个密码的推导过程，而非碰撞答案。
- 任一音频谜题都有视觉等价信息。
- 手机端无横向溢出，所有窗口最大化可用。
- 刷新保留已读状态和记事本，但不展示完成百分比。
- 结局前至少一次主动回看旧邮件。
